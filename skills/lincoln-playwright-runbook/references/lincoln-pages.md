# Lincoln 画面ごとの遷移・目的・成功条件

## 画面一覧

| 画面コード | 名称 | URL パターン | 使用する処理 |
|-----------|------|-------------|-------------|
| ログイン | ログイン画面 | `Ascsc1000InitAction.do` | 全処理の前提 |
| 2FA | 二要素認証画面 | (ログイン後リダイレクト) | 全処理の前提 |
| カレンダー詳細 | コピー元カレンダー | TBD | 処理0 |
| 6800 detail | 施設詳細 | TBD | 処理A |

## 各画面の詳細

### ログイン画面

- **目的**: Lincoln にログインする
- **遷移元**: ブラウザで直接アクセス
- **URL**: `https://www.tl-lincoln.net/accomodation/Ascsc1000InitAction.do`
- **操作**: ID/PW 入力 → ログインボタン
- **成功条件**: ログイン後の画面に遷移（2FA or ダッシュボード）
- **使用セレクタ**: `auth.loginIdInput`, `auth.loginPwInput`, `auth.loginButton`（全 TBD）

### 2FA 画面

- **目的**: 二要素認証を完了する
- **遷移元**: ログイン画面からリダイレクト
- **操作**: ユーザーが手動で 2FA コードを入力
- **Runner の動作**: `awaiting_user` 状態で待機 → ユーザー入力完了を検知 → 継続
- **タイムアウト**: 5 分（`TWO_FACTOR_TIMEOUT_MS` で設定可能）
- **成功条件**: 2FA 入力後のページ遷移を検知
- **使用セレクタ**: `auth.twoFactorInput`, `auth.twoFactorSubmit`（全 TBD）

### カレンダー詳細画面（処理0）

- **目的**: Excel の料金ランクを DOM 更新で注入 → 保存
- **遷移元**: ダッシュボード or URL 直接アクセス
- **操作**:
  1. DOM から月テーブル群を取得
  2. `page.evaluate` で hidden input を更新
  3. 保存ボタンをクリック
- **成功条件**: 保存成功レスポンスを検知
- **使用セレクタ**: `step0.*`（`saveButton` は TBD）

### 6800 detail 画面（処理A）

- **目的**: 操作対象の施設 ID が正しいことを確認する（安全弁）
- **遷移元**: カレンダー詳細 or URL 直接アクセス
- **操作**: `dl.g_header_id dd` から施設 ID を読み取り → 期待値と比較
- **成功条件**: 施設 ID が完全一致
- **失敗条件**: 不一致 → `FacilityMismatchError` で即停止
- **使用セレクタ**: `stepA.facilityIdText`

### 5050 画面（処理B）

- **目的**: 料金ランクを一括コピー
- **遷移元**: 処理A 完了後 or URL 直接アクセス
- **操作**:
  1. コピー元カレンダーをオートコンプリートで選択
  2. コピーボタン（`doCopy()`）をクリック
  3. 送信継続ボタン（`doSend(true)`）をクリック
  4. 全対象について繰り返し
- **成功条件**: 全カレンダーへのコピー→送信が正常完了
- **使用セレクタ**: `stepB.*`

### 5070 画面（処理C）

- **目的**: Lincoln の出力データと Excel 入力データを突合検証
- **遷移元**: 処理B 完了後 or URL 直接アクセス
- **操作**:
  1. 期間を設定（3ヶ月制約あり → 分割）
  2. プラングループを選択
  3. 「ランクのみ出力」トグルを有効化（TBD）
  4. 出力ボタンをクリック
  5. 出力データをダウンロード・解析
  6. Excel 入力データと突合
- **成功条件**: 全エントリで完全一致 → SUCCESS
- **失敗条件**: 1 件でも不一致 → FAILED、不一致詳細を artifacts に保存
- **使用セレクタ**: `stepC.*`（`rankOnlyToggle` は TBD）
- **制約**: 出力期間は最大 3ヶ月。超過時は分割して出力・検証
