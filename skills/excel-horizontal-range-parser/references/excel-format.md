# 入力 Excel の形式と終端判定

## ファイル名規約

```
【{施設名}様】料金変動案_{YYYYMMDD}[_自動生成].xlsx
```

例:
- `【畳の宿那覇壺屋様】料金変動案_20260201.xlsx`
- `【アクアパレス北谷様】料金変動案_20260215_自動生成.xlsx`

施設名は `【...様】` から正規表現で抽出:
```
/【(.+?)様】/
```

## シート構造

### ヘッダー行（4行目）

| C4 | D4 | E4 | F4 | ... | (空白) |
|----|-----|-----|-----|-----|--------|
| (起点) | 部屋タイプA | 部屋タイプB | 部屋タイプC | ... | ← 右端終端 |

### データ行（5行目〜）

| C列 | D列 | E列 | F列 | ... |
|------|------|------|------|------|
| 2026-04-01 | A | B | A | ... |
| 2026-04-02 | B | C | A | ... |
| 2026-04-03 | A | A | B | ... |
| ... | ... | ... | ... | ... |
| (空白) | ← 下端終端 |

### イメージ図

```
     A    B    C         D         E         F         G
  1  ...  ...  ...       ...       ...       ...       ...
  2  ...  ...  ...       ...       ...       ...       ...
  3  ...  ...  ...       ...       ...       ...       ...
  4  ...  ...  [起点]    部屋A     部屋B     部屋C     (空白)→終端
  5  ...  ...  2026-4-1  A         B         A
  6  ...  ...  2026-4-2  B         C         A
  7  ...  ...  2026-4-3  A         A         B
  :  ...  ...  :         :         :         :
  N  ...  ...  (空白)→終端
```

## 起点: C4

- **固定**: 常に C4 セルからデータ取得を開始する
- C4 自体は列ヘッダーの起点（通常は空白 or ラベル）
- 実際のデータヘッダーは D4 から始まる

## 右側終端判定

1. 4行目（ヘッダー行）を C4 から右方向に走査
2. セル値が `None` / 空文字 / 空白のみ のセルに到達したら終端
3. 終端セルの**1つ前の列**が右端

### 注意点
- 途中に空白セルがある場合は**終端と判定する**（途中の空白は許容しない）
- マージセルがある場合は openpyxl の `merged_cells` を考慮

## 下側終端判定

1. C列（日付列）を C5 から下方向に走査
2. セル値が `None` / 空文字 / 空白のみ のセルに到達したら終端
3. 終端セルの**1つ前の行**が下端

### 注意点
- 途中に空白行がある場合は**終端と判定する**
- 日付として解釈できない値がある場合は異常値として扱う

## 矩形領域

終端判定の結果、パース対象は以下の矩形:

```
左上: D5 (最初のデータセル)
右下: {右端列}{下端行}
列ヘッダー: D4 〜 {右端列}4
行ヘッダー: C5 〜 C{下端行}
```
