# 正規化ルールと禁止変換

## 正規化ルール（適用順）

### 1. 空白除去

| 対象 | 処理 |
|------|------|
| 前後の半角スペース | strip / trim |
| 前後の全角スペース（U+3000） | strip |
| セル内の改行（\n, \r\n） | 除去 |

### 2. 日付正規化（C列 = 行ヘッダー）

| 入力例 | 正規化後 | ルール |
|--------|---------|--------|
| `2026/4/1` | `2026-04-01` | YYYY-MM-DD 形式に統一 |
| `2026-4-1` | `2026-04-01` | ハイフン区切りも受付 |
| `2026年4月1日` | `2026-04-01` | 日本語形式も受付 |
| `4/1` | エラー | 年がない → 異常値 |
| Excel シリアル値 | `2026-04-01` | openpyxl が自動変換 |

### 3. ランクコード正規化（データセル）

| 対象 | 処理 |
|------|------|
| 前後の空白 | strip |
| 全角英数字 | 半角に変換（例: `Ａ` → `A`） |
| 小文字英字 | 大文字に変換（例: `a` → `A`） |

### 4. 部屋タイプ名正規化（4行目 = 列ヘッダー）

| 対象 | 処理 |
|------|------|
| 前後の空白 | strip |
| 全角スペース | 半角スペースに変換 |

## 禁止変換

以下の変換は**行わない**:

| 禁止事項 | 理由 |
|---------|------|
| ランクコードの推測補完 | 空白/null は異常値として扱う |
| 日付の推測補完 | 年が欠落した日付は異常値 |
| 部屋タイプ名の略称展開 | Lincoln 側の表記と一致しなくなる |
| 数値→文字列の暗黙変換 | openpyxl の型をそのまま使用 |
| マージセル内容の補完推測 | openpyxl のマージセル処理に従う |

## 異常値の扱い

### 停止（エラー）する場合

| 異常 | 理由 |
|------|------|
| C列の日付が解釈不能 | パース結果の信頼性がない |
| データセルに予期しない型（数値、bool 等） | ランクコードは文字列であるべき |
| C4 セルが存在しない / シートが存在しない | パースの前提が崩れている |

### 警告して続行する場合

| 異常 | 対処 |
|------|------|
| データセルが空白 | 警告ログ出力 + `null` として記録 |
| 重複する日付が存在 | 警告ログ出力 + 後勝ち（最後の値を採用） |
| ランクコードが 1 文字でない | 警告ログ出力 + そのまま記録 |

## 正規化の実装場所

- Python: `apps/runner/src/parsers/parse_excel.py` 内
- 正規化関数は個別にテスト可能な形で分離する
