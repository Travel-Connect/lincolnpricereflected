## 依頼内容

添付の Review Pack（ZIP）を確認し、以下の観点でレビューをお願いします。

このツールは、宿泊施設管理システム「リンカーン (TL-Lincoln)」に対し、
Excel の日別料金ランクを自動反映し、反映結果を出力 Excel と突合して成否を判定するものです。

### UI レビュー観点

1. **誤操作防止**: 破壊的操作（リンカーンへの送信・カレンダー更新）に確認ダイアログがあるか。undo が効かない操作に十分な警告があるか。特に「本番プラングループセット」（例：「〇単泊カレンダー」「□連泊カレンダー」）への誤送信を防ぐガード（テスト用セット以外は選択不可、または確認ダイアログ強化）はあるか。
2. **情報の優先順位**: 施設名・対象期間・ステータスなどの重要情報が視認しやすい位置にあるか。
3. **エラー表示**: エラー発生時に「次に何をすべきか」がユーザーに伝わるか。スタックトレースではなく、具体的な対処手順を示すメッセージが出るか。
4. **再開 (resume) の分かりやすさ**: ジョブが途中で中断（FAILED / CANCELLED）した場合の再開導線は明確か。
5. **2FA 導線**: 二段階認証が必要になった場合の案内は適切か。「AWAITING_2FA」ステータス時にユーザーが取るべきアクションが明確か。
6. **設定パターン UX**: カレンダーマッピング（入力Excel↔リンカーンカレンダー）・処理B設定（コピー元→プラングループセット）の入力は直感的で間違えにくいか。設定したマッピング内容が最終確認画面で十分に確認できるか。
7. **ジョブ一覧・履歴**: 過去の実行結果が効率的に検索・確認できるか。

### システムレビュー観点

1. **安全弁**: 施設IDの検証位置は適切か。意図しない施設への操作を防止できるか。`target_machine` による実行マシン制限は適切に機能するか。
2. **ログ/証跡の十分性**: 障害発生時に原因を特定するための情報（ログ、スクリーンショット、ネットワーク記録）は十分か。
3. **設定の永続化**: パターン設定（カレンダーマッピング等）が正しく保存・再利用されるか。施設ごとの設定が混在しないか。
4. **失敗時の復旧**: ジョブ失敗後のDB状態（status が RUNNING のまま残らないか）、ブラウザセッション（ゾンビプロセス）、ファイル（ロックファイル）の整合性は保たれるか。
5. **スキーマ/バケット整合**: DBマイグレーションと TypeScript 型定義は一致しているか。Storage バケット名のプレフィックスルールは守られているか。
6. **セッション管理**: Cookie の有効期限、2FA トークンの扱い、同一ユーザー多重セッションの排他制御。
7. **データ整合性**: Excel パース結果とリンカーンへの反映値が一致するか。突合ロジックの信頼性。
